1. what is the difference between .map .filter .forEach .find in javascript?

.map() method

The .map() method creates a new array with the results of calling a provided function on every element in the calling array. In other words, it returns a new array with the same length as the original array, but with each element transformed according to the provided function.

Example:

const numbers = [1, 2, 3, 4, 5];
const doubledNumbers = numbers.map((num) => num * 2);
console.log(doubledNumbers); // [2, 4, 6, 8, 10]

.filter() method

The .filter() method creates a new array with all elements that pass the test implemented by the provided function. In other words, it returns a new array with only the elements that meet the specified condition.

Example:

const numbers = [1, 2, 3, 4, 5];
const evenNumbers = numbers.filter((num) => num % 2 === 0);
console.log(evenNumbers); // [2, 4]

.forEach() method

The .forEach() method executes a provided function once for each array element. It doesn't return anything, but can be useful for performing a side effect on each element in the array.

Example:

const numbers = [1, 2, 3, 4, 5];
numbers.forEach((num) => console.log(num * 2));
// Output: 2 4 6 8 10

.find() method

The .find() method returns the value of the first element in the array that satisfies the provided testing function. If no values satisfy the testing function, undefined is returned.

Example:

const numbers = [1, 2, 3, 4, 5];
const evenNumber = numbers.find((num) => num % 2 === 0);
console.log(evenNumber); // 2

2. what is the difference between local Storage, session Storage and cookies in javascript ?

Local Storage:

  • Stores data with no expiration date
  • Data is stored on the client-side and persists even after the browser is closed
  • Can be accessed across different browser windows and tabs that share the same origin

Session Storage:

  • Stores data for the duration of a session
  • Data is stored on the client-side and is deleted when the browser is closed
  • Can be accessed across different browser windows and tabs that share the same origin

Cookies:

  • Stores small amounts of data with an expiration date
  • Data is stored on the client-side and persists even after the browser is closed if the expiration date hasn't passed
  • Can be accessed across different browser windows and tabs that share the same origin, as well as different domains if the cookie is set to be accessible to those domains

3. What is an event loop in JavaScript ?

An event loop is a constantly running loop in JavaScript that listens for asynchronous events and executes their corresponding callback functions when they are ready. By efficiently handling asynchronous code execution, the event loop improves the overall performance and responsiveness of JavaScript applications. Understanding how the event loop works is crucial for writing efficient and scalable code.

4. how does quizTimer() function work ?

The 'quizTimer' function is a timer that takes one argument called 'dismiss', which is a boolean. If 'dismiss' is true, the function clears the interval timer and stops the timer. Otherwise, it starts a new interval timer that runs every 1000ms (or 1 second).

Within the interval timer, the function updates the timer display on the HTML document by calculating the minutes and seconds and updating the 'innerHTML' of an element with the id 'count'. It also checks whether the count has gone over 60 seconds and alternates the text color between red-300 and red-700 every second.

The 'count' variable keeps track of the number of seconds passed since the timer started, and it gets incremented by one every second.

5. how does chooseQuiz() function work ?

The 'chooseQuiz' function takes two arguments: 'index' and 'givenAns'. The 'index' parameter is used to identify which quiz question is being answered, and 'givenAns' is the answer that the user has selected for that question.

The function first checks whether the user has already answered this question by searching through the 'answers' array to see if there is an existing answer object with the same 'id' as the current quiz question. If an existing answer object is found, the function updates the 'givenAns' property of that object with the user's current answer.

If an answer object is not found for the current quiz question, the function creates a new answer object and adds it to the 'answers' array with the current 'quizData[index]' and 'givenAns' properties.

Finally, the 'displayAnswers' function is called to update the UI and display the user's answers.
